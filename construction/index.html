Construction

Many web developers use version control systems, frameworks, extensions, and libraries to help them track changes and expedite web creation. This page goes into just what some of these technologies (and their licenses) can do.

Git

Track and share changes

Git is a "distributed version control system" (DVCS) that is "free and open source". Its primary function as a DVCS is to track changes made to files over time. These changes are stored in a repository that clients can mirror and backup if a server fails.

Bootstrap

Responsive made easy

Created in mid-2010, Bootstrap is currently the most popular front-end framework for fast and responsive web development. Users are free to use Bootstrap's code as long as they include its open-source license in “all copies or substantial portions”.

Sass

CSS made fun again

Sass is a nine-year-old CSS extension that "adds power and elegance to the basic language". Among other features, it allows the use of variables, nested rules, and mixins. Over the years, many frameworks, such as Compass, have been built with Sass.

jQuery

JavaScript made less infuriating

jQuery’s concept originated in 2005, where its founders state that it “started as an inkling”. Its purpose is to simplify JavaScript for web applications. Tasks normally requiring blocks of code are converted into simple keywords that users can call upon.
